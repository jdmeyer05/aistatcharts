AI STATCHARTS — UX IMPROVEMENTS CHANGELOG
==========================================
Date: March 19, 2026
Scope: 21 features across 25+ files, all 20 pages impacted


================================================================================
1. MOBILE SESSION RECOVERY — AUTO-RELOAD ON STALE CONNECTION
================================================================================
Files changed: src/layout.py

What changed:
  JavaScript is injected on every page that monitors when the phone goes to
  background. If the app was backgrounded for more than 60 seconds, the page
  automatically reloads when the user returns.

How it benefits users:
  Previously, mobile users returning to the app after locking their phone or
  switching apps would find all navigation buttons greyed out and unclickable.
  The only fix was to manually close and reopen the browser. Now the app
  silently recovers on its own — users pick up right where they left off.


================================================================================
2. PULL-TO-REFRESH BANNER
================================================================================
Files changed: src/layout.py

What changed:
  For shorter background periods (15–60 seconds) or after 5 minutes of idle
  time with no interaction, a yellow banner appears at the top of the screen:
  "Data may be stale — tap to refresh." Tapping it reloads the page.

How it benefits users:
  Users no longer have to guess whether the data on screen is current. The
  banner gives them a clear, low-friction way to refresh without hunting for
  a reload button or pulling down the browser address bar.


================================================================================
3. COOKIE-BASED AUTH PERSISTENCE
================================================================================
Files changed: src/auth.py, app.py

What changed:
  After login, the Supabase refresh token is stored in a browser cookie. When
  the session state is lost (server restart, long idle, mobile wake-up), the
  app reads the cookie and silently re-authenticates without showing the login
  screen.

How it benefits users:
  Mobile users previously had to re-enter their email and password every time
  the connection went stale. Now sessions survive app backgrounding, browser
  restarts, and even server redeployments. Users stay logged in seamlessly.


================================================================================
4. "REMEMBER ME" CHECKBOX
================================================================================
Files changed: app.py, src/auth.py

What changed:
  A "Remember me for 30 days" checkbox was added to the login form (checked
  by default). When checked, the auth cookie persists for 30 days. When
  unchecked, the cookie expires when the browser is closed.

How it benefits users:
  Users on shared or public devices can uncheck the box to ensure their
  session does not persist. Users on personal devices get the convenience
  of staying logged in for a month without re-authenticating.


================================================================================
5. LOGOUT BUTTON
================================================================================
Files changed: src/layout.py

What changed:
  A "Log Out" button was added to the sidebar on every page, along with a
  caption showing the signed-in email address. Logging out clears the auth
  cookie, wipes all session state, and redirects to the login screen.

How it benefits users:
  There was no way to sign out from within the app — users had to clear
  cookies manually. Now logout is one tap away from any page, and it
  properly cleans up all stored credentials.


================================================================================
6. SESSION TIMEOUT WARNING
================================================================================
Files changed: src/auth.py, app.py

What changed:
  The app tracks when the user authenticated. At 50 minutes, a toast
  notification warns: "Session expires soon — save your work." At 60 minutes,
  the app silently attempts to re-authenticate using the Supabase session or
  the stored cookie. If re-auth fails, the user is notified.

How it benefits users:
  Users running long AI analyses or reviewing results no longer get
  unexpectedly kicked to the login screen. They get advance warning and
  the app tries to renew the session automatically in the background.


================================================================================
7. RESPONSIVE NAVIGATION — MOBILE TOUCH TARGETS
================================================================================
Files changed: src/styles.py

What changed:
  Navigation buttons on phone screens now have a 44px minimum height (Apple's
  recommended tap target size), larger font, and generous padding. On screens
  under 640px, nav buttons wrap into rows of 3. On very small screens (under
  400px), they wrap into rows of 2 with 48px height.

How it benefits users:
  The 5-column nav bar was nearly unusable on phones — buttons were too small
  to tap accurately. Now every button is comfortably tappable even on small
  screens, and the layout adapts to the available width.


================================================================================
8. CACHED TICKER & ESCALATION DATA
================================================================================
Files changed: src/layout.py, src/ticker_tape.py

What changed:
  The escalation data reader (iran_conflict_history.json) was wrapped in
  @st.cache_data(ttl=300). The ticker tape data function now includes a
  _fetched_at timestamp for staleness detection.

How it benefits users:
  Every widget interaction previously re-read the JSON file from disk and
  re-fetched market data. Now these are cached for 5 minutes, making page
  interactions noticeably faster — especially on pages with many widgets.


================================================================================
9. SKELETON LOADERS & BATCH PREFETCH
================================================================================
Files changed: src/styles.py, src/layout.py, pages/01_Summary.py

What changed:
  A CSS shimmer animation and skeleton card component were added. The Summary
  page now prefetches all 12 market tickers upfront behind a loading skeleton,
  then renders all cards at once instead of one by one.

How it benefits users:
  On first load or cache expiry, the Summary page previously showed a blank
  screen for 10-20 seconds as cards trickled in one at a time. Now users see
  an animated loading indicator immediately, and all cards appear together
  once data is ready.


================================================================================
10. SMART AI TOKEN MANAGEMENT — NO BURN ON CACHE HITS
================================================================================
Files changed: pages/03_Stock_Analysis.py

What changed:
  AI model calls are cached for 30 minutes via @st.cache_data. Previously,
  clicking "Run Analysis" for the same ticker within that window still
  deducted a token even though the cached result was served. Now the app
  tracks a hash of the prompt — tokens are only deducted on genuinely
  fresh API calls.

How it benefits users:
  Users who re-run the same analysis (e.g., after navigating away and back)
  no longer waste their daily AI allowance or purchased tokens on cached
  results. This effectively stretches their quota further.


================================================================================
11. STALE DATA INDICATOR
================================================================================
Files changed: src/layout.py, src/ticker_tape.py

What changed:
  A red "DELAYED 23m" badge appears in the header when market data is more
  than 15 minutes old. The ticker strip timestamp is color-coded: green for
  fresh (<5 min), yellow for warming (5–15 min), red with a warning icon
  for stale (>15 min).

How it benefits users:
  Users making trading decisions need to know if the data on screen is
  current. The visual indicators make staleness immediately obvious,
  preventing decisions based on outdated prices.


================================================================================
12. AUTO-REFRESH DURING MARKET HOURS
================================================================================
Files changed: src/layout.py

What changed:
  The ticker strip is wrapped in @st.fragment(run_every=300), causing it to
  automatically re-render every 5 minutes. This triggers a fresh data fetch
  (the cache TTL also aligns at 5 minutes). The fragment updates without
  reloading the entire page.

How it benefits users:
  During market hours, users no longer need to manually refresh to see
  updated prices. The ticker strip updates itself in the background every
  5 minutes while the rest of the page remains interactive and unchanged.


================================================================================
13. RECENTLY VISITED PAGES
================================================================================
Files changed: src/layout.py

What changed:
  A "Recent:" row appears below the navigation bar showing the last 5 pages
  the user visited (excluding the current page). Each entry is a clickable
  link with the page icon.

How it benefits users:
  With 20 pages, users typically rotate between 4-5 favorites. The recent
  pages row saves them from opening dropdown menus to find pages they were
  just on. One tap to jump back.


================================================================================
14. DEEP LINKING — TICKER FOLLOWS YOU ACROSS PAGES
================================================================================
Files changed: src/layout.py, 11 page files

What changed:
  A shared active ticker system was added. When a user types "AAPL" on any
  page, it is stored in session state. When they navigate to another page,
  the ticker input pre-fills with "AAPL" instead of the default "SPY."
  URL deep linking is also supported via ?ticker=TSLA query parameter.

Pages updated: Stock Analysis, Historical Analysis, Options Analysis,
  Options Flow, Options Lab, ML Predictor, Tech Screener, Algo Backtester,
  Monte Carlo, RL Trading, Scenario Analysis (partial).

How it benefits users:
  Previously, switching from Stock Analysis to Options Flow for the same
  ticker meant retyping it. Now the ticker carries over automatically.
  Users can also share links like ?ticker=NVDA that open directly to
  their desired asset.


================================================================================
15. BREADCRUMB CONTEXT
================================================================================
Files changed: src/layout.py

What changed:
  The header bar now shows a breadcrumb: "AI STATCHARTS > Stock Analysis > AAPL"
  displaying the current page name and the active ticker being analyzed.

How it benefits users:
  After switching between several pages and tickers, users can glance at
  the header to instantly confirm what page they are on and which asset
  they are viewing. This is especially helpful on mobile where less
  context is visible.

16. ERROR BOUNDARIES ON ALL SECTIONS
================================================================================
Files changed: src/layout.py, src/styles.py, 14 page files

What changed:
  - Options Analysis: all 9 tabs individually wrapped (Vol Surface, IV Skew,
    Term Structure, Open Interest, Gamma Exposure, Max Pain, Unusual Activity,
    Greeks Heatmap, Chain View).
  - Iran Conflict: all 3 tabs wrapped.
  - 13 additional pages now import error_boundary for section-level use.
  - Streamlit's default exception display was restyled to match the app's
    dark theme with a red border and scrollable traceback.
  - A page_error_boundary() helper was added for wrapping entire pages.

How it benefits users:
  Previously, a single API failure or data error would crash an entire page,
  showing an ugly Python traceback. Now each section fails independently —
  if the IV Skew tab errors out, the other 8 tabs still render normally.
  Error messages are styled clearly with instructions to refresh.

               pages/02_Scenario_Analysis.py

How it benefits users:
  Users previously could double-click or triple-click the "Run Analysis"

  API calls. Now the button clearly communicates that work is in progress

19. DAILY BRIEFING
================================================================================
Files changed: pages/01_Summary.py

What changed:
  A new "Daily Briefing" section appears near the top of the Summary page,
  generating a 3-5 sentence market summary. It synthesizes:
  - Live market data: equity index direction, VIX fear gauge, commodities
  - Cached AI analysis: Grok macro regime + market sentiment
  - Geopolitical risk: Iran conflict escalation (when elevated)
  The briefing is styled in an accent-bordered card and costs zero API
  calls — it is built entirely from already-cached data.

How it benefits users:
  Users get an instant, human-readable snapshot of market conditions the
  moment they open the app. No need to scan multiple charts or navigate
  to different pages — the briefing distills the key information into a
  few sentences. Especially valuable for mobile users who want a quick
  check-in.


================================================================================
20. WATCHLIST WITH PRICE ALERTS
================================================================================
Files changed: pages/01_Summary.py

What changed:
  A new "Watchlist" section on the Summary page allows users to:
  - Add tickers with optional price thresholds (alert above $X, below $X).
  - View mini price cards for each watchlist ticker (price + daily change).
  - Remove tickers with a single click.
  - Receive automatic alerts when:
    * Price crosses above or below their set thresholds.
    * Any watchlist ticker moves more than 5% in a day.
  Watchlist alerts appear in the existing Alerts section alongside regime
  shift and VIX alerts. The watchlist persists across page navigations
  within the same session.

How it benefits users:
  Users can track specific assets they care about without navigating to
  individual analysis pages. The price threshold alerts surface important
  moves automatically — users do not need to manually check prices. This
  turns the Summary page into a personalized command center.


================================================================================
21. STYLED EXCEPTION DISPLAY
================================================================================
Files changed: src/styles.py

What changed:
  Streamlit's default exception/traceback display (.stException) was restyled
  with a dark background, red border, rounded corners, and a scrollable,
  smaller-font traceback section.

How it benefits users:
  When errors do occur, they no longer look like a jarring white-background
  code dump. They match the app's visual design and are easier to read,
  with the traceback contained in a scrollable area that does not dominate
  the screen.

Mobile Experience:

Navigation:

  - Recent pages are one tap away
  - Breadcrumbs show current context at all times

Reliability:
  - Error boundaries prevent one bad section from crashing a page
  - Session timeout warnings prevent unexpected logouts
  - AI buttons prevent double-click token waste

Engagement:
  - Daily briefing gives instant market context on login
  - Watchlist with alerts turns the Summary into a command center
  - Usage gauge helps users pace their AI allowance
  - Account dashboard shows session and subscription status
